# 数据库设计

## 一、数据库选型

| 场景 | 数据库 | 用途 |
|------|--------|------|
| **桌面/本地模式** | SQLite | 轻量级，零配置，单文件存储 |
| **云端/企业模式** | PostgreSQL | 高并发，支持事务，企业级可靠性 |
| **向量存储（本地）** | ChromaDB Embedded | 嵌入式运行，无需单独部署 |
| **向量存储（生产）** | PGVector | PostgreSQL 扩展，统一运维 |
| **缓存/队列** | Redis | Token 黑名单、限流、Celery 消息队列 |

> 关系数据库使用 **SQLAlchemy Async ORM**，通过异步引擎（`aiosqlite` / `asyncpg`）实现非阻塞数据库操作。

### 3.2 知识管理核心

```mermaid
classDiagram
    class KnowledgeBase {
        +int id PK
        +int user_id FK → User
        +string name
        +text description
        +int embedding_model_id FK → ModelConfig
        +int doc_count
        +int chunk_count
        +string chunk_strategy [fixed|paragraph|recursive|heading]
        +int chunk_size 默认500
        +int chunk_overlap 默认50
        +string search_mode [vector|keyword|hybrid]
        +int context_window 默认1
        +text welcome_message
        +text prompt_template
        +datetime created_at
        +datetime deleted_at 软删除
    }

    class Document {
        +int id PK
        +int kb_id FK → KnowledgeBase
        +string filename
        +string file_path
        +bigint file_size
        +string file_type
        +int chunk_count
        +enum status [uploading|parsing|embedding|completed|failed]
        +text error_message
        +string content_hash
        +datetime expires_at
        +bool is_archived
        +datetime created_at
        +datetime deleted_at 软删除
    }

    class DocumentChunk {
        +int id PK
        +int doc_id FK → Document
        +int kb_id FK → KnowledgeBase
        +text content 切片原文
        +int chunk_index 序号
        +text metadata JSON元数据
        +datetime created_at
    }

    KnowledgeBase "1" --> "*" Document
    Document "1" --> "*" DocumentChunk
    KnowledgeBase "1" --> "*" DocumentChunk
```

> **注意**：`DocumentChunk` 同时冗余了 `kb_id`，便于检索时直接按知识库过滤，避免 JOIN。

### 3.3 对话系统

```mermaid
classDiagram
    class ChatConversation {
        +int id PK
        +int user_id FK → User
        +int kb_id FK → KnowledgeBase
        +string title
        +int llm_model_id FK → ModelConfig
        +int total_input_tokens
        +int total_output_tokens
        +text context_summary 语义摘要缓存
        +datetime created_at
    }

    class ChatMessage {
        +int id PK
        +int conversation_id FK → ChatConversation
        +string role [user|assistant|system]
        +text content
        +text references JSON引用来源
        +int token_count
        +string feedback [good|bad]
        +string feedback_reason
        +float latency_ms 响应延迟
        +string msg_type [chat|compaction]
        +datetime created_at
    }

    ChatConversation "1" --> "*" ChatMessage
```

**设计要点**：
- `total_input/output_tokens`：追踪每个对话的 Token 消耗量
- `context_summary`：语义摘要引擎的缓存，避免重复压缩
- `latency_ms`：记录每条回复延迟，用于性能监控
- `msg_type = compaction`：标记被压缩合并的历史消息

### 3.4 模型配置

```mermaid
classDiagram
    class ModelConfig {
        +int id PK
        +int user_id FK → User
        +enum model_type [llm|embedding|reranker]
        +enum provider [openai|anthropic|ollama|custom]
        +string api_base API地址
        +string api_key_encrypted 加密存储
        +string model_name 模型标识
        +string display_name 显示名称
        +text params JSON参数
        +bool is_default
        +text fallback_model_ids JSON降级模型列表
        +datetime created_at
    }
```

**安全设计**：API Key 使用 AES-256 加密存储（`api_key_encrypted`），调用时解密。

### 3.5 工作空间（多租户）

```mermaid
classDiagram
    class Workspace {
        +int id PK
        +string name
        +string description
        +int owner_id FK → User
        +datetime created_at
    }

    class WorkspaceMember {
        +int id PK
        +int workspace_id FK
        +int user_id FK
        +enum role [owner|admin|member|viewer]
        +datetime joined_at
    }

    class WorkspaceKnowledgeBase {
        +int id PK
        +int workspace_id FK
        +int kb_id FK UK
        +datetime created_at
    }

    class WorkspaceModelConfig {
        +int id PK
        +int workspace_id FK
        +int model_config_id FK
        +int shared_by FK → User
        +datetime created_at
    }

    class WorkspaceInvitation {
        +int id PK
        +int workspace_id FK
        +string invite_token UK
        +enum role
        +int max_uses
        +int use_count
        +bool is_active
        +datetime expires_at
    }

    Workspace "1" --> "*" WorkspaceMember
    Workspace "1" --> "*" WorkspaceKnowledgeBase
    Workspace "1" --> "*" WorkspaceModelConfig
    Workspace "1" --> "*" WorkspaceInvitation
```

**多租户隔离模型**：
- 不采用 schema-per-tenant，而是通过 **关联表** 实现资源共享
- 一个知识库只能属于一个工作空间（`kb_id` 有 UK 约束）
- 成员角色四级：Owner > Admin > Member > Viewer

## 五、关键索引设计

| 表 | 索引 | 用途 |
|----|------|------|
| `users` | `ix_username`, `ix_email` | 登录查找 |
| `documents` | `ix_documents_kb_status` (kb_id, status) | 按知识库+状态筛选文档 |
| `document_chunks` | `ix_chunks_kb_doc` (kb_id, doc_id) | 检索时按知识库快速查找切片 |
| `chat_conversations` | `ix_conv_user_created` (user_id, created_at) | 用户对话列表排序 |
| `workspace_members` | `uq_workspace_member` (workspace_id, user_id) | 防止重复加入 |
